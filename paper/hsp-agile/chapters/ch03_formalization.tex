\chapter{Formal Problem Definition}
\label{ch:formalization}

Release under a conjunctive gate needs a small set of precise objects:
a typed task, a notation-agnostic SpecIR, a first-match oracle with
specification-derived witnesses, and an acceptance predicate under budget~$K$.
This chapter defines those objects; Chapter~\ref{ch:method} turns them into a
runnable loop.
Notation: $\mathbb{N}$, $\mathbb{Z}$, and $\mathbf{1}[\cdot]$ for the Iverson
bracket.

Throughout, the \textbf{primary} running example is the catch-all tier
classifier of Chapter~\ref{ch:intro}
(Listings~\ref{lst:others-fsf}--\ref{lst:others-bug}): busy Reject/Pass tests
can clear while the residual \texttt{OTHERWISE} band still returns
\texttt{Pass} instead of \texttt{Review}.
The industrial-monitoring contract \code{classify\_signal} and the ordered tax
brackets of Listing~\ref{lst:tax-fsf} appear only as \textbf{secondary}
illustrations of guard overlap and priority witnesses~$\Phi_i$.

\section{Formal Task Model}
\label{sec:form:task}
% ---------------------------------------------------------------------------

A release decision needs a precise unit of work: one function whose behaviour
is described entirely by an ordered scenario specification.
That unit is a \emph{formal specification task}.
HSP-Agile instantiates the model over SOFL/FSF~\citep{li2022qrs_companion,liu2004soflbook};
every later stage consumes one such task.

\begin{definition}[Formal Specification Task]
\label{def:task}
A \emph{formal specification task} is a triple
\[
  t \;=\; \bigl(\mathcal{S}_t,\;\mathcal{X}_t,\;\mathcal{Y}_t\bigr),
\]
where
\begin{itemize}
  \item $\mathcal{S}_t = \langle s_1, s_2, \ldots, s_n \rangle$ is a finite,
        ordered sequence of $n$ FSF scenarios (the \emph{specification});
  \item $\mathcal{X}_t \subseteq \mathbb{Z}^d$ is the \emph{input domain},
        a $d$-dimensional integer vector space; and
  \item $\mathcal{Y}_t \subseteq \mathbb{Z}^m$ is the \emph{output domain},
        an $m$-dimensional integer vector space.
\end{itemize}
\end{definition}

\paragraph{Primary example (catch-all tier).}
For \code{classify\_tier}, $d=2$, $m=1$ (string-valued tier encoded as a
discrete label), $\mathcal{S}_t=\langle s_1,s_2,s_3\rangle$, and
$\mathcal{X}_t\subseteq\mathbb{Z}^2$ with coordinates
$(\mathit{score},\mathit{floor})$.
Benchmark tasks in later chapters often use $d=5$ and $m=3$; the same
definitions apply.

\section{Specification Intermediate Representation (SpecIR)}
\label{sec:form:specir}
% ---------------------------------------------------------------------------

The prevention pipeline should not be rewritten for every surface notation.
What is needed is a shared hinge: a notation-agnostic form that still preserves
ordered guarded cases---\textbf{semantic regions}, not surface syntax.
That hinge is the \textbf{Specification Intermediate Representation}
(\textbf{SpecIR}).
Adapters (Chapter~\ref{ch:intro}; implementation in Chapter~\ref{ch:method})
map SOFL/FSF, Mini-Z, or Mini-StateMachine text into SpecIR; after that step
the shared pipeline never sees the original syntax.
Witness generation, conformance, and acceptance all consume SpecIR cases as
first-match regions.

\begin{definition}[Specification Intermediate Representation]
\label{def:specir}
For task $t$, $\text{SpecIR}(t) = (\Sigma_t,\; \Gamma_t,\; C_t,\; M_t)$, where
$\Sigma_t$ is the typed I/O signature, $\Gamma_t$ the variable domain,
$C_t = \langle c_1,\ldots,c_n\rangle$ an ordered list of guarded cases
(first-match semantics), and $M_t$ notation-specific metadata.
The abstract syntax is written
$\text{SpecIR} \triangleq \langle\Sigma_t, \Gamma_t, C_t, M_t\rangle$ or,
equivalently, $\text{SpecIR} ::= \texttt{Signature}\;\texttt{Cases}\;[\texttt{Metadata}]$.
\end{definition}

\paragraph{Primary example.}
For \code{classify\_tier}, $\Sigma_t$ declares inputs
\texttt{score:int}, \texttt{floor:int} and output \texttt{tier:string};
$C_t$ holds three guarded cases, ending in an \texttt{others} residual whose
postcondition is the constant \texttt{Review}.
Each case is one FSF scenario $s_i=(g_i,\phi_i)$: a Boolean guard
$g_i:\mathcal{X}_t\to\{0,1\}$ and a total output mapping
$\phi_i:\mathcal{X}_t\to\mathcal{Y}_t$
(Definition~\ref{def:scenario}).

\begin{definition}[SpecIR Surface Grammar (EBNF)]
\label{def:specir-ebnf}
The JSON serialisation
(\texttt{schemas/spec\_ir.schema.json}, \texttt{src/ir/spec\_ir.py})
is $\texttt{SpecIR}::=\texttt{task\_id}\;\texttt{notation}\;\texttt{name}\;
\texttt{Signature}\;\texttt{Cases}\;[\texttt{Metadata}]$, with
$\texttt{GuardedCase}::=\texttt{index}\;\texttt{guard}\;\texttt{postcondition}$
and $\texttt{GuardAtom}::=\texttt{var}\;\texttt{op}\;[\texttt{threshold}]$
($\texttt{op}\in\{\texttt{eq},\texttt{ne},\texttt{lt},\texttt{le},\texttt{gt},
\texttt{ge},\texttt{others}\}$).
\end{definition}

\begin{definition}[SpecIR Well-formedness]
\label{def:specir-wf}
A SpecIR document $d$ is \emph{well-formed} iff:
(i)~$\texttt{notation}\in\{\texttt{sofl},\texttt{mini\_z},
\texttt{mini\_statemachine},\texttt{real\_derived}\}$;
(ii)~case indices are consecutive $1,\ldots,n$ with pairwise distinct guards;
(iii)~every guard $\texttt{var}$ (except $\texttt{others}$) is declared in
\texttt{inputs}, and every postcondition key in \texttt{outputs}; and
(iv)~the final case is $\texttt{others}$ or equivalent to
$\neg\bigvee_{i<n} g_i$.
Enforced by \texttt{src/ir/schema\_validate.py}
(Appendix~\ref{sec:repro:schemas}).
\end{definition}

\begin{definition}[Specification Adapter]
\label{def:spec-adapter}
$\text{Adapt}_L : \text{Text}_L \longrightarrow \text{SpecIR}$
(Chapter~\ref{ch:method}).
\end{definition}

\begin{definition}[FSF Lowering (HSP-Agile Instantiation)]
\label{def:fsf-lowering}
$\text{Lower}_{\text{FSF}} : \text{SpecIR} \longrightarrow \text{TaskSpec}_{\text{FSF}}$
projects SpecIR onto an FSF-shaped view for screening; the pipeline core
depends only on SpecIR.
\end{definition}

\begin{definition}[FSF Scenario]
\label{def:scenario}
$s_i = (g_i,\;\phi_i)$ with $g_i : \mathcal{X}_t \to \{0, 1\}$ and
$\phi_i : \mathcal{X}_t \to \mathcal{Y}_t$.
\end{definition}

Priority is part of the specification, not an implementation
detail~\citep{dijkstra1975guarded}.
The oracle is the function that enforces it.

\begin{definition}[Scenario Ordering and Specification Oracle]
\label{def:oracle}
Scenario $s_i$ has \emph{higher priority} than $s_j$ whenever $i < j$.
For any input $x \in \mathcal{X}_t$, the \emph{specification oracle}
$g_t : \mathcal{X}_t \to \mathcal{Y}_t$ is
\[
  g_t(x) \;=\; \phi_i(x),
  \qquad \text{where } i = \min\bigl\{j : g_j(x) = 1\bigr\}.
\]
The final scenario $s_n$ is the \emph{``others'' case} with
$g_n(x) = \neg\bigvee_{j=1}^{n-1} g_j(x)$, so every input is handled by
exactly one scenario.
\end{definition}

\paragraph{Primary example.}
On $(\mathit{score},\mathit{floor})=(3,10)$, neither $g_1$
($\mathit{score}<0$) nor $g_2$ ($\mathit{score}\ge\mathit{floor}$) holds, so
$g_t(3,10)=\phi_3(3,10)=\texttt{Review}$.
An implementation that collapses the residual to \texttt{Pass}
(Listing~\ref{lst:others-bug}) disagrees with the oracle on every mid-band
input, while still matching on busy Reject and Pass samples.

\paragraph{Secondary example (ordering / overlap).}
On \code{classify\_signal} at $(-3,-10)$, both $g_1$ ($\mathit{level}<0$) and
$g_2$ ($\mathit{level}\ge\mathit{threshold}$) hold, so
$g_t(-3,-10)=\phi_1(-3,-10)=\texttt{"Error"}$.
An implementation that checks $g_2$ first returns \texttt{"Critical"} and
fails the oracle wherever both guards hold---correct on many disjoint inputs,
wrong on the overlap.

\section{The Bounded Acceptance Problem}
\label{sec:form:acceptance}
% ---------------------------------------------------------------------------

The oracle answers what a correct implementation must return.
Release is stronger: match every specification-derived witness \emph{and} clear
a structural screen, within a fixed LLM budget.
We state the acceptance objects first; Section~\ref{sec:form:semantics}
then constructs the witnesses that make conformance operational.

\begin{definition}[Candidate Implementation]
\label{def:candidate}
Any computable $c : \mathcal{X}_t \to \mathcal{Y}_t$; write $f_c$ for its
observable behaviour.
\end{definition}

\begin{definition}[Strict Formal Conformance]
\label{def:conformance}
Given a finite case set $\mathcal{D}(t)\subseteq\mathcal{X}_t\times\mathcal{Y}_t$
(Definition~\ref{def:test-suite}),
\[
  \mathrm{Conf}(c,\,t)
  \;=\;
  \frac{1}{|\mathcal{D}(t)|}
  \sum_{(x,\,y)\,\in\,\mathcal{D}(t)}
  \mathbf{1}\bigl[f_c(x) = y\bigr]
  \;\in\; [0,1].
\]
\emph{Strict success} (Definition~\ref{def:strict-success}) is
$\mathrm{Conf}(c,t)\ge\tau$ with $\tau=1$; a lower $\tau$ would mask residual
or ordering violations that pass most cases where the omitted region is
thin~\citep{zhu1997specification}.
\end{definition}

\paragraph{Primary example.}
If $\mathcal{D}(t)$ has three witnesses---one Reject, one Pass, one
\texttt{others}---and the buggy \code{classify\_tier} fails only on the
residual, then $\mathrm{Conf}(c,t)=2/3$.

\begin{definition}[Strict Success]
\label{def:strict-success}
$\mathrm{Conf}(c,\,t) \ge \tau$ with $\tau = 1$.
\end{definition}

Conformance alone is still insufficient: structural anti-patterns can clear
every witness.
The high-severity screen $\mathcal{P}^H$ (Table~\ref{tab:patterns};
Appendix~\ref{app:patterns}) blocks those cases.

\begin{definition}[High-Severity Pattern Violation]
\label{def:pattern-violation}
$c$ is \emph{pattern-safe} iff
$\forall p \in \mathcal{P}^H : p(c,\,t) = 0$.
\end{definition}

Release is the conjunction of the two checks---neither alone is enough.
This is the lead claim of the report:
$\mathrm{Accept}\Leftrightarrow\mathrm{Conf}{=}1\wedge\mathrm{Screen}{=}\mathit{pass}$.

\begin{definition}[Hybrid Acceptance Predicate]
\label{def:accept}
\[
  \mathrm{Accept}(c,\,t)
  \;=\;
  \mathbf{1}\!\left[
    \mathrm{Conf}(c,\,t) \geq 1
    \;\land\;
    \forall p \in \mathcal{P}^H : p(c,\,t) = 0
  \right].
\]
\end{definition}

\paragraph{Primary example.}
A repaired \code{classify\_tier} with $\mathrm{Conf}=1$ still fails
acceptance if RF07 (\emph{unconditional success}) fires on a hard-coded
\texttt{return "Pass"}.
Conversely, clearing the pattern screen while failing the \texttt{others}
witness also fails $\mathrm{Accept}$.
Mean conformance can be high while acceptance stays selective
(Chapter~\ref{ch:results}).

Those pieces assemble into the computational problem: find an acceptable
candidate within budget~$K$, or return the best-effort candidate when the
budget is exhausted.
Semantic feedback on attempt~$k{+}1$ is a structured JSON record
(counterexamples, pattern hits, attempt index); Chapter~\ref{ch:method}
details its fields.
The search is CEGIS-shaped~\citep{solarlezama2008sketching,sun2024clover}
with a stochastic synthesiser---hence the explicit fallback.

\begin{definition}[Bounded Acceptance Problem]
\label{def:bap}
Given task $t$, budget $K \in \mathbb{N}^+$, and synthesis oracle
$\mathrm{LLM}(\cdot)$, find
$c^* = \mathrm{LLM}(t,\mathrm{feedback}_1,\ldots,\mathrm{feedback}_{k-1})$
with at most $K$ calls such that $\mathrm{Accept}(c^*, t) = 1$;
otherwise return
$c^* = \arg\max_{c \in C_K} \mathrm{Conf}(c,\,t)$
where $C_K$ is the set of candidates generated in the $K$ attempts.
\end{definition}

\section{Witnesses and Scenario Semantics}
\label{sec:form:semantics}
% ---------------------------------------------------------------------------

The oracle in Definition~\ref{def:oracle} is a mathematical object.
Conformance (Definition~\ref{def:conformance}) needs concrete inputs on which
to execute candidates.
Those inputs are SMT-generated witnesses rather than random samples---one
representative per strict first-match region, including the residual
\texttt{others} band.

\begin{definition}[Concrete Case]
\label{def:concrete-case}
A \emph{concrete case} is $(x,\,g_t(x))$ for $x\in\mathcal{X}_t$;
the \emph{case set} is $\mathcal{D}(t)$.
\end{definition}

\begin{definition}[SMT-Generated Witness Suite]
\label{def:test-suite}
For each scenario $s_i = (g_i, \phi_i)$, solve
\[
  x_i \;\models\; g_i(x) \;\land\; \bigwedge_{j < i} \neg\,g_j(x)
\]
with Z3~\citep{demoura2008z3, barrett2018smt}; if satisfiable, add
$\bigl(x_i,\;g_t(x_i)\bigr)$ to $\mathcal{D}(t)$.
Equivalently, writing
$\Phi_i(x) \triangleq g_i(x) \land \bigwedge_{j<i}\neg g_j(x)$
(Equation~\eqref{eq:phi-intro}; Chapter~\ref{ch:method},
Equation~\eqref{eq:priority}), each $x_i$ is a model of~$\Phi_i$.
The conjunct $\bigwedge_{j < i} \neg\,g_j(x)$ enforces strict scenario
membership: $x_i$ activates $s_i$ and no higher-priority scenario, so each
case is a direct witness to $\phi_i$.
For the final \texttt{others} case, $\Phi_n$ is exactly the residual region
missed by busy-band unit tests in the primary vignette.
\end{definition}

\paragraph{Primary example.}
For $S_3$ (\texttt{others}) on \code{classify\_tier}, the query is
$\neg(\mathit{score}<0)\land\neg(\mathit{score}\ge\mathit{floor})$; a model
might be $(3,10)$ with oracle label \texttt{Review}.
Reject and Pass witnesses alone cannot expose Listing~\ref{lst:others-bug}.

\paragraph{Secondary example (\code{compute\_tax}).}
For $S_1$ the query is $\mathit{income}<0$; a model might be $(-1,0)$ with
oracle bracket \texttt{Invalid}.
For $S_2$ (foreign-high),
$\Phi_2 \equiv (\mathit{income}{\ge}120000 \wedge \mathit{is\_foreign}{=}1)
\land \neg(\mathit{income}{<}0)$; a model is $(150000,1)$ with oracle
\texttt{ForeignHigh}---the overlap slice drawn in
Figure~\ref{fig:defect-vs-test}.

\begin{lemma}[Scenario Coverage]
\label{lem:coverage}
If the SMT solver finds satisfying inputs for all $n$ scenarios, then
$\mathcal{D}(t)$ contains at least one representative case per scenario,
covering the entire precedence ordering $s_1 \succ s_2 \succ \cdots \succ s_n$.
\end{lemma}

\begin{proof}
By construction, each $x_i$ satisfies $g_i$ but not $g_1, \ldots, g_{i-1}$.
Hence $i = \min\{j : g_j(x_i) = 1\}$, so the oracle evaluates
$g_t(x_i) = \phi_i(x_i)$.  Each distinct scenario $s_i$
contributes at least one case whose correct output is determined by
$\phi_i$, and the precedence ordering is witnessed by all $n$ cases jointly.
\end{proof}

Residual and precedence defects concentrate near thin regions of the input
space---catch-all bands and guard boundaries.
Random tests rarely land there; when they miss, unit-test repair looks
healthy while residual or ordering bugs remain.
The corollary below contrasts random sampling with SMT targeting.

\begin{definition}[Boundary Region]
\label{def:boundary-region}
For margin $\delta > 0$,
$\mathcal{B}_\delta(t) = \{\mathbf{x} \in \mathcal{X}_t :
  \exists i,\; \partial g_i(\mathbf{x}) < \delta\}$.
\end{definition}

\begin{definition}[Per-witness Fault-Exposure Probability]
\label{def:fault-exposure}
For a faulty candidate $c$,
\[
  \varepsilon_c \;=\;
  \Pr_{\mathbf{x} \sim \mathcal{U}(\mathcal{X}_t)}
  \bigl[\,f_c(\mathbf{x}) \neq g_t(\mathbf{x})\,\bigr],
\]
and $\varepsilon_{\min} = \min_{c \in \mathcal{C}\setminus\{g_t\}}
\varepsilon_c > 0$ when the faulty class $\mathcal{C}$ is nonempty.
\end{definition}

\begin{corollary}[Random vs.\ SMT Fault-Exposure Hit Rate]
\label{cor:random-vs-smt}
Let
$\rho = \mu(\mathcal{B}_\delta(t))/\mu(\mathcal{X}_t)$
denote the relative measure of the boundary (or residual) region.
Assume faults of interest are supported only on
$\mathcal{B}_\delta(t)$.
Then a single uniform random sample detects such a fault with probability at
most~$\rho$.
The Constraint-guided Witness Generator produces one witness per strict
scenario region via $\Phi_i$ (Equation~\eqref{eq:priority}); under a faithful
SMT encoding, each such witness activates scenario~$i$ with probability~$1$
(Proposition~\ref{thm:witness-soundness}).
Random sampling reaches precedence-critical or residual regions
with probability $\mathcal{O}(\rho)$, whereas SMT achieves targeted coverage
of each $\Phi_i$ deterministically.
\end{corollary}

\begin{proof}
Under the support assumption and uniform sampling,
\[
  \Pr\bigl[f_c(\mathbf{x})\neq g_t(\mathbf{x})\bigr]
  \;\le\;
  \Pr\bigl[\mathbf{x}\in\mathcal{B}_\delta(t)\bigr]
  \;=\;\rho.
\]
For SMT, satisfiability of $\Phi_i$ yields
$g_i(\mathbf{x}_i^*)\land\bigwedge_{j<i}\neg g_j(\mathbf{x}_i^*)$, so
$\mathbf{x}_i^*$ lies in the exclusive first-match region of~$s_i$ and
activates that region with probability~$1$ under a faithful encoding
(Proposition~\ref{thm:witness-soundness}).
Hence random sampling hits the critical regions with probability
$\mathcal{O}(\rho)$, while SMT hits them deterministically.
\end{proof}

\begin{definition}[Strict Scenario Violation]
\label{def:violation}
Candidate $c$ \emph{violates} $s_i$ on $(x_i, y_i) \in \mathcal{D}(t)$ if
$f_c(x_i) \neq y_i = g_t(x_i)$; the \emph{counterexample} is
$(x_i,\,g_t(x_i),\,f_c(x_i))$.
\end{definition}

\paragraph{Primary example.}
For Listing~\ref{lst:others-bug} on $(3,10)$, the counterexample is
$\bigl((3,10),\;\texttt{Review},\;\texttt{Pass}\bigr)$ and the violated
scenario is $s_3$ (\texttt{others}).
Counterexamples name the active scenario, so the repair model can localise
the bug inside the ordered specification
(Feedback IR fields in Chapter~\ref{ch:method}).

\section{Prevention and Quality Metrics}
\label{sec:form:metrics}
% ---------------------------------------------------------------------------

Generation success is the wrong primary metric for a prevention system.
What matters is whether faulty candidates are kept out of release, and whether
correct ones are not rejected by accident---both measured over a mutant
population~\citep{demillo1978mutation,ammann2016testing}.

\begin{definition}[Mutant Population]
\label{def:mutants}
$\mathcal{M}$ is a finite set of pairs $(c_m, t_m)$ with at least one side
perturbed: $\mathcal{M}^S$ mutates the task, $\mathcal{M}^I$ the candidate;
$\mathrm{Faulty}(m)=1$ when the pair is genuinely faulty.
\end{definition}

Let $\mathcal{M}^F = \{m \in \mathcal{M} : \mathrm{Faulty}(m)=1\}$ be the
faulty-mutant subset.
Both prevention metrics use $|\mathcal{M}^F|$ as denominator:
PDR is the fraction of faulty mutants correctly rejected;
FAR is the escape rate among faulty mutants (false-accept rate),
\emph{not} a precision analogue~\citep{okun2013sate4}.

\begin{definition}[Prevention Detection Rate]
\label{def:pdr}
\[
  \mathrm{PDR}
  \;=\;
  \frac{\bigl|\bigl\{m \in \mathcal{M}^F : \neg\,\mathrm{Accept}(c_m,\,t_m)\bigr\}\bigr|}
       {|\mathcal{M}^F|}.
\]
\end{definition}

\begin{definition}[False Accept Rate]
\label{def:far}
\[
  \mathrm{FAR}
  \;=\;
  \frac{\bigl|\bigl\{m \in \mathcal{M}^F :
      \mathrm{Accept}(c_m,\,t_m) = 1
    \bigr\}\bigr|}
       {|\mathcal{M}^F|}.
\]
\end{definition}

On a fixed faulty-mutant set, $\mathrm{PDR}+\mathrm{FAR}=1$.

\begin{definition}[Mutation Kill Rate]
\label{def:mkr}
$\mu_k = |\{m\in\mathcal{M}:\mathrm{Conf}(c_m,t_m)<1\}|/|\mathcal{M}|$;
the gap $\mathrm{PDR}-\mu_k$ is the additive value of pattern screening.
\end{definition}

\section{Benchmark Formalization}
\label{sec:form:benchmark}
% ---------------------------------------------------------------------------

The evaluation uses a synthetic benchmark of hard tasks built to stress FSF
precedence sensitivity~\citep{li2023iet_faultprevention}, including residual
\texttt{others} cases and ordered overlaps.

\begin{definition}[Precedence-Sensitive Task]
\label{def:prec-sensitive}
$t$ is \emph{precedence-sensitive} if there exist $s_i,s_j$ with $i<j$ and
overlapping guards:
$\{x:g_i(x)=1\}\cap\{x:g_j(x)=1\}\neq\emptyset$.
On the overlap, output depends on evaluation order whenever
$\phi_i$ and $\phi_j$ disagree---the class of error naive LLM synthesis
frequently introduces on secondary ordering vignettes, and a distinct failure
mode from the primary catch-all mismatch.
\end{definition}

\begin{proposition}
\label{prop:order-required}
Let $t^H$ be precedence-sensitive, and suppose there exist $s_i,s_j$ with
$i<j$ and an input $x^*$ in their guard overlap such that
$\phi_i(x^*)\neq\phi_j(x^*)$.
Then any implementation that evaluates $g_j$ before $g_i$ fails strict
conformance on~$x^*$.
\end{proposition}

\begin{proof}
By Definition~\ref{def:oracle}, $g_t(x^*)=\phi_i(x^*)$.
If $c$ evaluates $g_j$ before $g_i$ and $g_j(x^*)=1$, then
$f_c(x^*)=\phi_j(x^*)$.
The hypothesis $\phi_i(x^*)\neq\phi_j(x^*)$ yields
$f_c(x^*)\neq g_t(x^*)$, so $\mathrm{Conf}(c,t^H)<1$ whenever
$\mathcal{D}(t^H)$ includes a witness for the exclusive region of~$s_i$
that contains~$x^*$ (or $x^*$ itself).
\end{proof}

Hard tasks $t^H$ use five non-negative inputs, three integer outputs
($\mathit{risk},\mathit{action},\mathit{score}$), up to $n=9$ ordered
scenarios, and seven random thresholds $\theta_1,\ldots,\theta_7\in[1,100]$;
Z3 feasibility and a scenario-consistency validator discard unsatisfiable or
ill-covered $\theta$ vectors.
By Proposition~\ref{prop:order-required}, every hard task with a
semantically relevant overlap requires ordered guard evaluation; SMT
witnesses catch order-insensitive LLM outputs.
Residual \texttt{others} scenarios are covered by the same $\Phi_i$
construction (Definition~\ref{def:test-suite}).

\section{Computational Considerations}
\label{sec:form:complexity}
% ---------------------------------------------------------------------------

Formal checking must not erase the prevention benefit by dominating runtime.
With $n\le 9$ and $d=5$, Z3 case generation is
$O(n\cdot d^2)$ and finishes in $\sim$1\,s; conformance is $O(n)$ lookups;
pattern screening is $O(L\cdot|\mathcal{P}^H|)$ and negligible
($\ll$0.01\,s)~\citep{demoura2008z3, barrett2018smt}.
The dominant cost is the LLM call ($\sim$10\,s/attempt): with $K=3$,
expected runtime is $\sim$30\,s/task, of which $>97\%$ is LLM inference.
In short,
$T_{\mathrm{pipeline}}\approx K\cdot T_{\mathrm{LLM}}+O(n\cdot d^2)\,T_{\mathrm{Z3}}
+O(n)\,T_{\mathrm{exec}}+O(L\cdot P)\,T_{\mathrm{pattern}}$
