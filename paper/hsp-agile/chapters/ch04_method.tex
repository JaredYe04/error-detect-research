\chapter{The \textsc{SgDP} Framework and HSP-Agile Instantiation}
\label{ch:method}

This chapter presents the methodological foundation of HSP-Agile at two levels.
\textbf{Part~A} (Section~\ref{sec:method:sgdp}) introduces the
\textbf{Specification-guided Defect Prevention Framework} (\textsc{SgDP}), its
instantiation interface, three formal theorems, the Semantic Feedback IR, the
conjunctive acceptance predicate, prevention metrics, and the full algorithm.
\textbf{Part~B} (Sections~\ref{sec:method:principles}--\ref{sec:method:summary})
details the four-stage pipeline: design principles P1--P5, per-stage algorithms,
prompt engineering, and the requirement pattern guard.
We inherit the notation of Chapter~\ref{ch:formalization}:
$t = (\mathcal{S}_t, \mathcal{X}_t, \mathcal{Y}_t)$, oracle $g_t$, and
first-match semantics over $\mathcal{S}_t = \langle s_1,\ldots,s_n\rangle$.

\section{The \textsc{SgDP} Framework}
\label{sec:method:sgdp}

\subsection{Framework Overview and Instantiation Interface}
\label{subsec:framework-overview}

The \textbf{Specification-guided Defect Prevention Framework} (\textsc{SgDP})
prevents specification-conformance defects in LLM-generated implementations via
four language-independent components:
\begin{enumerate}
  \item \textbf{Specification Oracle} ($\mathcal{G}$): executable oracle derived
    from the formal specification without a reference implementation.
  \item \textbf{Constraint-guided Witness Generator} ($\mathcal{W}$): SMT-based
    witnesses targeting boundary regions where conformance defects manifest.
  \item \textbf{Scenario-aware Semantic Feedback IR} ($\mathcal{F}$): typed
    failure records carrying specification-level semantics for repair prompts.
  \item \textbf{Conjunctive Acceptance Predicate} ($\mathcal{A}$): release gate on
    oracle-grounded conformance and specification-derived quality criteria.
\end{enumerate}

Figure~\ref{fig:sgdp-overview} (Chapter~\ref{ch:intro}) depicts the end-to-end
\textsc{SgDP} pipeline; the present section details the instantiation interface.

An instantiation for language $\mathcal{L}$ provides:
(i)~Specification Adapter $\text{Adapt}_L : \text{Text}_L \to \text{SpecIR}$;
(ii)~oracle $g_t$ derived from $\text{SpecIR}(t).\text{cases}$ under first-match;
(iii)~witness generator $\mathcal{W}(t)$ from SpecIR guard formulas;
(iv)~optional lowerer $\text{Lower}_{\text{FSF}}$ (HSP-Agile) $\to$ FSF TaskSpec;
(v)~Semantic Feedback IR per \texttt{schemas/semantic\_feedback.schema.json},
    rendered by $\rho : \mathcal{F} \to \text{Prompt}$.
Synthesis, repair, pattern guard, and acceptance are shared core components.

\subsection{HSP-Agile: SOFL/FSF Instantiation}
\label{subsec:hsp-agile}

HSP-Agile instantiates \textsc{SgDP} over FSF~\cite{liu2014sofl,li2022agilesofl}.
Each scenario $s_i = (g_i, \phi_i)$ pairs a guard with a postcondition; under
\emph{first-match semantics} the output for $\mathbf{x}$ is $\phi_i(\mathbf{x})$ where
$i = \min\{j : g_j(\mathbf{x})\}$.
The FSF oracle requires no reference implementation:
\begin{equation}
  g_t(\mathbf{x}) = \phi_i(\mathbf{x}) \text{ where } i = \min\{j : g_j(\mathbf{x})\}.
\end{equation}

\subsection{Constraint-guided Witness Generation}
\label{subsec:witness-gen}

For scenario $s_i$, the witness generator solves the \emph{priority formula}
\begin{equation}
  \label{eq:priority}
  \Phi_i \;\triangleq\; \hat{g}_i(\mathbf{x}) \;\land\; \bigwedge_{j=1}^{i-1} \lnot\hat{g}_j(\mathbf{x}),
\end{equation}
where $\hat{g}_j$ is the Z3 encoding of $g_j$, ensuring activation of $s_i$ under
first-match semantics.
The witness set is $\mathcal{D}(t) = \{\mathbf{x}_1^*, \ldots, \mathbf{x}_n^*\}$.

\begin{theorem}[Scenario-aware Witness Soundness]
\label{thm:witness-soundness}
If $\hat{g}_i$ soundly encodes $g_i$ and Z3 returns model $\mathbf{x}^*$ satisfying
$\Phi_i$, then $g_i(\mathbf{x}^*)$ holds and $g_j(\mathbf{x}^*)$ is false for all $j < i$;
hence $g_t(\mathbf{x}^*) = \phi_i(\mathbf{x}^*)$.
\end{theorem}
\begin{proof}
Z3 soundness and encoding correctness give $g_i(\mathbf{x}^*)$ and $\lnot g_j(\mathbf{x}^*)$
for $j < i$; first-match semantics yields $i = \min\{j : g_j(\mathbf{x}^*)\}$.
(Proof sketch.)
\end{proof}

\subsection{Scenario-aware Semantic Feedback IR}
\label{subsec:feedback-ir}

On witness failure, the framework records
\begin{equation}
  \mathcal{F}(i, c, t) = \langle \textit{type},\ s_i,\ g_i,\ \mathbf{x}_i^*,
  \ c(\mathbf{x}_i^*),\ g_t(\mathbf{x}_i^*),\ \textit{reason},\
  \textit{priority},\ \textit{fix} \rangle,
\end{equation}
with \textit{type} $\in \{\text{ordering}, \text{boundary}, \text{arithmetic},
\text{output}\}$, ranked \textit{priority}, and suggested \textit{fix}.
A template renders this IR into repair prompts naming the violated scenario, stating
the guard, and enforcing first-match ordering.
The IR serialises to JSON per \texttt{schemas/semantic\_feedback.schema.json};
\texttt{FeedbackRenderer} (\texttt{src/repair/feedback\_ir.py}) projects the same
IR onto E6 render variants (test-only, test+expected, full semantic IR), demonstrating
IR--prompt separation rather than ad-hoc prompt engineering.

\subsection{Formal Conformance and Acceptance Predicate}
\label{subsec:conformance-acceptance}

\begin{equation}
  \label{eq:conformance}
  \mathrm{Conf}(c,t) = \frac{1}{|\mathcal{D}(t)|} \sum_{\mathbf{x} \in \mathcal{D}(t)}
  \mathbf{1}\bigl[f_c(\mathbf{x}) = g_t(\mathbf{x})\bigr].
\end{equation}
Let $\mathcal{P}^H$ be the nine high-severity RF patterns (Table~\ref{tab:patterns}).
The conjunctive acceptance predicate is:
\begin{equation}
  \label{eq:accept}
  \mathrm{Accept}(c,t) \;\iff\;
  \mathrm{Conf}(c,t) = 1 \;\land\;
  \sum_{p \in \mathcal{P}^H} \mathbf{1}[p(c,t)] = 0.
\end{equation}

\begin{theorem}[Finite-Sample FAR Bound]
\label{thm:far-bound}
For a task $t$ with $|\mathcal{D}(t)| = n$ SMT-generated witnesses drawn
independently, the false-acceptance rate satisfies
$\mathrm{FAR} \leq (1-\varepsilon)^n$,
where $\varepsilon > 0$ is the minimum per-witness fault-exposure probability
(Definition~\ref{def:fault-exposure}).
The bound holds under the assumption that witnesses are independently generated
and not adversarially chosen.  It does not constitute full correctness: a
candidate accepted under~$\mathrm{Accept}(c,t)$ may still fail on inputs outside
$\mathcal{D}(t)$.
\end{theorem}
\begin{proof}
Independence of witnesses gives $P(\text{all pass} \mid \text{faulty}) \leq
(1-\varepsilon)^n$ by the union bound complement.  Witness soundness
(Theorem~\ref{thm:witness-soundness}) ensures each witness $\mathbf{x} \in
\mathcal{D}(t)$ exercises a non-trivial guard boundary, making $\varepsilon$
bounded away from zero for the fault classes in the mutation operators
(Table~\ref{tab:operators}); Corollary~\ref{cor:random-vs-smt} strengthens
this argument by placing witnesses on precedence-critical boundaries.
(Proof sketch.)
\end{proof}

\subsection{Structured Feedback and Repair Hypothesis}
\label{subsec:repair-monotonicity}

\begin{hypothesis}[Expected Improvement under Structured Feedback]
\label{hyp:structured-feedback}
Under fixed model and budget $K$, repair guided by Semantic Feedback IR
(which names the violated scenario, guard predicate, and violation type)
achieves higher expected conformance gain per iteration than repair guided
by plain test-pass/fail feedback.
\end{hypothesis}
\begin{evidence}
Empirical support is provided by E5 (repair convergence trajectories,
Fig.~\ref{fig:convergence}) and E6 (feedback variant comparison,
Fig.~\ref{fig:feedback-variant}): mode~M with structured IR achieves
$\mu_k$ convergence in fewer iterations than B2 (test-feedback) on the
same task set.  A fully formal proof would require a stochastic model of
LLM repair that is beyond current theory; we state this as an empirically
supported hypothesis.
\end{evidence}

\subsection{Empirical FAR Validation}
\label{subsec:far-validation}

To ground the Finite-Sample FAR Bound (Theorem~\ref{thm:far-bound})
empirically, we compare the theoretical upper bound $(1-\varepsilon)^n$ to
the observed FAR measured in E2 (Section~\ref{sec:e2-prevention}).
Using $\varepsilon = 0.08$ (estimated from the median per-witness detection
probability of implementation-level mutants) and $n = 9$ (average witness
count per task), the bound predicts $\mathrm{FAR} \leq (0.92)^9 \approx 0.473$.
The empirically measured FAR on mode~M in E2 is $0.14$ (Table~\ref{tab:far-validation}),
well within the theoretical bound.

\begin{inlinetable}{Theoretical FAR bound vs.\ empirical FAR (mode M, E2).}
  \label{tab:far-validation}
  \footnotesize
  \begin{tabular}{lrr}
    \toprule
    Mode & Empirical FAR & Bound $(1-\varepsilon)^n$ \\
    \midrule
    B1  & 0.38 & --- \\
    B2  & 0.22 & --- \\
    M   & 0.14 & 0.47 \\
    \bottomrule
  \end{tabular}
\end{inlinetable}

The empirical FAR (0.14) is substantially below the theoretical bound (0.47),
which is expected since the bound is conservative: witnesses are targeted at
boundary regions rather than drawn uniformly at random, making the per-witness
detection probability $\varepsilon$ higher in practice than the conservative
estimate used in the bound.

\subsection{Prevention-Oriented Metrics}
\label{subsec:metrics}

For mutant set $\mathcal{M}$:
\begin{equation}
  \mathrm{PDR} = \frac{|\{m \in \mathcal{M} : \neg\mathrm{Accept}(c_m, t_m)\}|}{|\mathcal{M}|},
  \qquad
  \mathrm{FAR} = \frac{|\{m \in \mathcal{M} : \mathrm{Accept}(c_m, t_m) \land \text{faulty}(m)\}|}{|\mathcal{M}|}.
\end{equation}
We report PDR, FAR, strict formal conformance, and latency jointly.

\subsection{Pipeline Complexity Cost Model}
\label{subsec:complexity-cost}

Let $n = |\mathcal{S}_t|$ denote the number of ordered scenarios in
$\text{SpecIR}(t)$ and $K$ the repair budget.
Witness generation solves $n$ priority formulas $\Phi_i$, each a conjunction of
$O(i)$ guard literals, giving per-task witness cost
$\mathcal{O}\bigl(\sum_{i=1}^{n} T_{\text{SMT}}(|\Phi_i|)\bigr)$ where
$T_{\text{SMT}}$ is solver time on linear integer arithmetic.
Formal checking executes $|\mathcal{D}(t)| \leq n$ concrete cases per
candidate; repair multiplies LLM and checker cost by up to~$K$.
The dominant term in mode~M is therefore
$\mathcal{O}(K \cdot (T_{\text{LLM}} + n \cdot T_{\text{exec}}))$ with
$T_{\text{SMT}}$ amortised once per task.
E3--E4 empirically confirm that this cost scales sub-linearly in conformance
gain: overlap density increases $n$ and tightens $\mathcal{B}_\varepsilon(t)$,
widening the random-vs.-SMT hit-rate gap of Corollary~\ref{cor:random-vs-smt}.

\subsection{Full Algorithmic Procedure}
\label{sec:method:algorithm}

\begin{algorithm}[t]
\caption{HSP-Agile (\textsc{SgDP} instantiation)}
\label{alg:sgdp}
\begin{algorithmic}[1]
\Require task $t$, budget $K$
\Ensure accepted candidate or best-effort fallback
\State $\mathcal{D}(t) \leftarrow \mathcal{W}(t)$
\State $\text{feedback}_0 \leftarrow \emptyset$; $\text{best} \leftarrow \mathtt{null}$
\For{$k = 1$ to $K$}
  \State $c_k \leftarrow \Fn{LLM}(t, \text{feedback}_{k-1})$
  \State $\mathrm{Conf}(c_k,t),\ \text{failures}_k \leftarrow$ eval $c_k$ on $\mathcal{D}(t)$ via $g_t$
  \State $\text{patterns}_k \leftarrow$ pattern guard on $c_k$
  \If{$\mathrm{Accept}(c_k, t)$} \Return $c_k$, accepted \EndIf
  \State $\mathcal{J}_k \leftarrow \textsc{ToJson}(\mathcal{F}(\text{failures}_k))$
        \Comment{schema: \texttt{semantic\_feedback.schema.json}}
  \State $\text{feedback}_k \leftarrow \rho(\mathcal{J}_k)$
        \Comment{\texttt{FeedbackRenderer} projection (E6 variants)}
  \State $\text{feedback}_k \mathrel{+}= \text{top-}5$ pattern violations from $\text{patterns}_k$
  \State update $\text{best}$ if $\mathrm{Conf}(c_k,t) > \mathrm{Conf}(\text{best},t)$
  \State log $\langle k, \mathrm{Conf}(c_k,t), |\text{failures}_k|, |\text{patterns}_k|\rangle$
\EndFor
\Return $\text{best}$, not-accepted
\end{algorithmic}
\end{algorithm}

Every iteration appends $\langle k, \mathrm{Conf}(c_k,t), |\text{failures}_k|,
|\text{patterns}_k|\rangle$ to \texttt{attempt\_history} in the output JSONL,
supporting repair-dynamics (E5) and feedback-quality (E6) analyses.

\section{Design Principles}
\label{sec:method:principles}

Five principles shaped every architectural decision~\citep{woodcock2009formalsurvey, chen2021codex}:

\paragraph{P1 --- Precision over approximation.}
Conformance is decided by an SMT solver and oracle $g_t$, not learned heuristics
or hand-annotated expected values.

\paragraph{P2 --- Bounded budget with guaranteed termination.}
A hard budget $K$ caps synthesis-check iterations ($K = 5$ in experiments).

\paragraph{P3 --- Counterexample specificity via Semantic Feedback IR.}
Repair feedback is constructed as JSON-serialised Semantic Feedback records
(\texttt{schemas/semantic\_feedback.schema.json}) and \emph{then} projected to
natural language; each record carries failing input~$\mathbf{x}^*$, oracle output
$g_t(\mathbf{x}^*)$, candidate output $f_c(\mathbf{x}^*)$, violated scenario index,
guard text, violation type, and optional fix hint.

\paragraph{P4 --- Conservative acceptance.}
Release requires both formal conformance and pattern safety
(equation~\eqref{eq:accept}); neither stage alone suffices.

\paragraph{P5 --- Modularity and ablation transparency.}
Boolean flags enable or disable Stages~2--4 independently for ablation
(Chapter~\ref{ch:experiments}).

\label{sec:method:synthesis}

Stage~1 maps $(t, \mathit{feedback}_{k-1})$ to candidate Python $c_k$; it is the

a fixed \emph{system} message plus a \emph{user} message carrying $(\mathcal{S}_t,
\mathit{sig}(t))$ and, when $k > 1$, $(\mathit{cexs}_{k-1}, \mathit{hits}_{k-1})$.

\subsection{Model Configuration and Parsing}
\label{subsec:method:model}

We use Qwen2.5-27B-Instruct at $\tau = 0.2$ (max 2048 output tokens).
The parser extracts one \code{```python} fenced block, compiles it, and on failure
sets $c_k = \bot$ (conformance 0).
\begin{equation}
  c_k \;=\; \mathrm{LLM}\!\left(t,\;\mathit{feedback}_{k-1}\right).
  \label{eq:synthesis}

\section{Stage 2: SMT-Driven Formal Conformance Checking}
\label{sec:method:checker}

Stage~2 computes $\mathrm{Conf}(c_k, t)$ and counterexamples $\mathit{cexs}_k$
via four steps~\citep{demoura2008z3, barrett2018smt} (Algorithm~\ref{alg:checker}):
\textbf{(1)} translate each guard $g_i$ to Z3 $\hat{g}_i$ and form $\Phi_i$
(equation~\eqref{eq:priority});
\textbf{(2)} solve $\Phi_i$ per scenario (Listing~\ref{lst:z3-gen});
\textbf{(3)} compute oracle outputs $y_i^* = g_t(x_i)$ by substituting into $\phi_i$;
\textbf{(4)} execute $c_k$ in a sandbox and compare field-by-field with type normalisation.

\inputpython{listings/z3_gen.py}{Z3 test-case generation for scenario~$i$.}{lst:z3-gen}

\begin{algorithm}[t]
\caption{SMT-Driven Formal Conformance Checking}
\label{alg:checker}
\begin{algorithmic}[1]
\Require task $t$, candidate $c$
\Ensure $\mathrm{Conf}(c,t) \in [0,1]$, counterexample list $\mathit{cexs}$
\State $\mathcal{D}(t) \leftarrow \emptyset$; $\mathit{cexs} \leftarrow \emptyset$
\For{$i = 1$ \textbf{to} $n$}
  \State $\Phi_i \leftarrow \ProcCall{TranslateToZ3}{s_i,\, \{g_j\}_{j<i}}$
  \If{$\ProcCall{Z3-Solve}{\Phi_i} = \mathit{sat}$}
    \State $x_i \leftarrow \ProcCall{ExtractModel}{}$; $y_i^* \leftarrow \phi_i(x_i)$
    \State $\mathcal{D}(t) \leftarrow \mathcal{D}(t) \cup \{(x_i, y_i^*)\}$
  \EndIf
\EndFor
\For{\textbf{each} $(x, y^*) \in \mathcal{D}(t)$}
  \State $y^c \leftarrow f_c(x)$
  \If{$y^c \neq y^*$}
    \State $\mathit{cexs} \leftarrow \mathit{cexs} \cup
           \{(x, y^*, y^c, \ProcCall{ScenarioOf}{x})\}$
  \EndIf
\EndFor
\State \Return $(|\mathcal{D}(t)| - |\mathit{cexs}|)/|\mathcal{D}(t)|,\, \mathit{cexs})$
\end{algorithmic}
\end{algorithm}

\subsection{Why SMT Over Random Testing}
\label{subsec:method:smt-rationale}

Random or boundary-value sampling rarely hits multi-conjunct guard regions
\citep{claessen2000quickcheck, zhu1997specification}.
$\Phi_i$ guarantees each witness lies in scenario $i$'s exclusive first-match
region, realising Theorem~\ref{thm:witness-soundness} and eliminating oracle ambiguity.

\section{Stage 3: Counterexample-Guided Repair}
\label{sec:method:repair}

When $\mathrm{Conf}(c_k, t) < 1$ or pattern violations occur, Stage~3 assembles
$\mathit{fb}_k$ mapping $\mathit{cexs}_k$, $\mathit{hits}_k$, attempt $k$, and
$\mathrm{Conf}(c_k,t)$~\citep{lynn2024verifierloop, solarlezama2008sketching}.
Each counterexample $(x^*, y^*, y^c, i^*)$ is verbalised, e.g.:

\begin{quote}
  \textit{``For inputs $(a{=}15,\; b{=}8,\; c{=}3,\; d{=}0,\; e{=}0)$, your code returned
  \{\textit{risk}{=}\code{MEDIUM}, \textit{action}{=}\code{REVIEW}\}, but the
  specification requires
  \{\textit{risk}{=}\code{HIGH}, \textit{action}{=}\code{ESCALATE}\}.
  This input satisfies Scenario~S1 ($a > 10$ AND $b > 5$), which should be matched first.''}
\end{quote}

form $\mathcal{P}^H$ used in equation~\eqref{eq:accept}.

\subsection{Detection Method}
\label{subsec:method:detection}

Each pattern $p \in \mathcal{P}$ has detector $\delta_p : \mathcal{C} \times \mathcal{T} \to \{0,1\}$.
Structural patterns (RF04, RF07, RF09, RF13) use AST analysis;
surface patterns (RF03, RF08, RF14) use regex;
guard-comparison patterns (RF01, RF02, RF05, RF06, RF11) cross-reference FSF guards
with candidate conditionals.
The hit set is $\mathrm{Hits}(c,t) = \{p \in \mathcal{P} : \delta_p(c,t) = 1\}$.
Medium/low-severity hits inform feedback but do not block acceptance.
