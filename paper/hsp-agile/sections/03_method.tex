\section{The \textsc{SgDP} Framework and HSP-Agile Instantiation}
\label{sec:method}

\subsection{Framework Overview}
\label{subsec:framework-overview}

The \textbf{Specification-guided Defect Prevention Framework} (\textsc{SgDP}) is a
four-component architecture for preventing specification-conformance defects in
LLM-generated implementations.  Its design is independent of any specific specification
language; a concrete instantiation binds each abstract component to a language-specific
implementation.

\begin{enumerate}
  \item \textbf{Specification Oracle} ($\mathcal{G}$): derives a ground-truth executable
    oracle directly from the formal specification, without requiring a human-written
    reference implementation.

  \item \textbf{Constraint-guided Witness Generator} ($\mathcal{W}$): uses constraint
    solving to produce semantically targeted test witnesses that navigate to the boundary
    regions of the input space where specification-conformance defects manifest.

  \item \textbf{Scenario-aware Semantic Feedback IR} ($\mathcal{F}$): structures
    conformance failures as a typed intermediate representation carrying
    specification-level information, enabling repair prompts grounded in formal semantics
    rather than test-case symptoms.

  \item \textbf{Conjunctive Acceptance Predicate} ($\mathcal{A}$): gates release on both
    oracle-grounded conformance and specification-derived quality criteria,
    providing a machine-checkable and human-auditable acceptance decision.
\end{enumerate}

\paragraph{Instantiation interface.}
An instantiation of \textsc{SgDP} for specification language $\mathcal{L}$ provides:
(i) a parser mapping $\mathcal{L}$ text to a \texttt{TaskSpec} IR;
(ii) an oracle function $g_t: \mathcal{X}_t \to \mathcal{Y}_t$ derived from
\texttt{TaskSpec};
(iii) a witness generator $\mathcal{W}(t) \subseteq \mathcal{X}_t$;
(iv) a feedback renderer mapping \texttt{SemanticFeedback} records to natural-language
repair prompts.
All other framework logic — the synthesis loop, repair loop, pattern guard invocation,
and acceptance predicate evaluation — is shared across instantiations.

\subsection{HSP-Agile: SOFL/FSF Instantiation}
\label{subsec:hsp-agile}

HSP-Agile instantiates \textsc{SgDP} over the SOFL Functional Specification Format
(FSF)~\cite{liu2004soflbook,li2022qrs_companion}.
An FSF task $t = (\mathcal{S}_t, \mathcal{X}_t, \mathcal{Y}_t)$ where
$\mathcal{S}_t = \langle s_1, \ldots, s_n \rangle$ is an ordered scenario sequence;
each scenario $s_i = (g_i, \phi_i)$ pairs a guard predicate $g_i$ with a postcondition
$\phi_i$.  FSF uses \emph{first-match semantics}: the output for input $\mathbf{x}$ is
$\phi_i(\mathbf{x})$ where $i$ is the smallest index with $g_i(\mathbf{x})$ true.

\paragraph{Oracle.}  The FSF oracle is:
\begin{equation}
  g_t(\mathbf{x}) = \phi_i(\mathbf{x}) \text{ where } i = \min\{j : g_j(\mathbf{x})\}.
\end{equation}
No reference implementation is required; the oracle is derived directly from the spec.

\subsection{Constraint-guided Witness Generation and Theorem 1}
\label{subsec:witness-gen}

For each scenario $s_i$, the witness generator must find $\mathbf{x}^*$ satisfying
the \emph{priority formula}:
\begin{equation}
  \label{eq:priority}
  \Phi_i \;\triangleq\; \hat{g}_i(\mathbf{x}) \;\land\; \bigwedge_{j=1}^{i-1} \lnot\hat{g}_j(\mathbf{x}),
\end{equation}
where $\hat{g}_j$ is the Z3 encoding of guard $g_j$.  This formula ensures the witness
activates $s_i$ under first-match semantics, not merely satisfies $g_i$ in isolation.
The witness set $\mathcal{D}(t) = \{\mathbf{x}_1^*, \ldots, \mathbf{x}_n^*\}$ contains
one witness per scenario.

\begin{proposition}[Scenario-aware Witness Soundness]
\label{thm:witness-soundness}
Let $\hat{g}_i$ be a sound Z3 encoding of $g_i$ (i.e., $\hat{g}_i(\mathbf{x}) \iff
g_i(\mathbf{x})$ for all $\mathbf{x}$).  If Z3 returns a model $\mathbf{x}^*$
satisfying $\Phi_i$, then $g_i(\mathbf{x}^*)$ is true and $g_j(\mathbf{x}^*)$ is false
for all $j < i$.  Consequently, $g_t(\mathbf{x}^*) = \phi_i(\mathbf{x}^*)$ — the
oracle assigns $\mathbf{x}^*$ to scenario $i$ under first-match semantics.
Soundness assumes a faithful SMT encoding of SpecIR guards.
\end{proposition}
\begin{proof}[Sketch]
By Z3 soundness, if $\mathbf{x}^*$ satisfies $\Phi_i$ then $\hat{g}_i(\mathbf{x}^*)$
and $\lnot\hat{g}_j(\mathbf{x}^*)$ for $j < i$.  By encoding correctness,
$g_i(\mathbf{x}^*)$ and $\lnot g_j(\mathbf{x}^*)$ for $j < i$.  By first-match
semantics, $i = \min\{j : g_j(\mathbf{x}^*)\}$, so the oracle evaluates via $\phi_i$.
\end{proof}

Proposition~\ref{thm:witness-soundness} establishes that every witness is semantically
valid under that encoding: conformance failures on witnesses are genuine violations, not
artefacts of test-input construction.

\subsection{Scenario-aware Semantic Feedback IR}
\label{subsec:feedback-ir}

When a candidate fails on witness $\mathbf{x}_i^*$, the framework constructs a
\texttt{SemanticFeedback} record rather than a raw test failure:

\begin{equation}
  \mathcal{F}(i, c, t) = \langle \textit{type},\ s_i,\ g_i,\ \mathbf{x}_i^*,
  \ c(\mathbf{x}_i^*),\ g_t(\mathbf{x}_i^*),\ \textit{reason},\
  \textit{priority},\ \textit{fix} \rangle,
\end{equation}

where \textit{type} $\in \{\text{ordering}, \text{boundary}, \text{arithmetic},
\text{output}\}$ classifies the violation;
\textit{reason} is an inferred natural-language explanation of the root cause;
\textit{priority} ranks violations by semantic severity (ordering errors ranked highest);
and \textit{fix} suggests a structural code change category.

The IR is rendered to a natural-language repair prompt via a template that explicitly
names the violated scenario, states the guard constraint in human-readable form, and
directs the LLM to respect the first-match ordering.  Section~\ref{sec:results} (E6)
shows that this structure produces faster convergence than test-case-only or
test-plus-expected feedback.

\subsection{Formal Conformance and Acceptance Predicate}
\label{subsec:conformance-acceptance}

The formal conformance score over the witness set is:
\begin{equation}
  \label{eq:conformance}
  \mathrm{Conf}(c,t) = \frac{1}{|\mathcal{D}(t)|} \sum_{\mathbf{x} \in \mathcal{D}(t)}
  \mathbf{1}\bigl[f_c(\mathbf{x}) = g_t(\mathbf{x})\bigr].
\end{equation}

Let $\mathcal{P}^H$ be the high-severity pattern set (9 of the 14 RF patterns whose
violation indicates a requirement anti-pattern incompatible with FSF semantics).
The conjunctive acceptance predicate is:
\begin{equation}
  \label{eq:accept}
  \mathrm{Accept}(c,t) \;\iff\;
  \mathrm{Conf}(c,t) = 1 \;\land\;
  \sum_{p \in \mathcal{P}^H} \mathbf{1}[p(c,t)] = 0.
\end{equation}

\begin{theorem}[Acceptance Safety]
\label{thm:acceptance-safety}
If $\mathrm{Accept}(c,t)$, then $c$ passes all witnesses in $\mathcal{D}(t)$ and
exhibits no high-severity requirement anti-pattern.  The false-accept rate (FAR) under
oracle $g_t$ is bounded by the probability that a faulty implementation happens to agree
with $g_t$ on all $|\mathcal{D}(t)|$ witnesses; for $|\mathcal{D}(t)| = n$ scenarios
with independent witnesses, $\mathrm{FAR} \leq (1-\epsilon)^n$ where $\epsilon$ is the
minimum probability that a randomly sampled witness exposes the fault.
\end{theorem}
\begin{proof}[Sketch]
$\mathrm{Accept}(c,t)$ requires $\mathrm{Conf}(c,t)=1$, which requires $f_c(\mathbf{x})
= g_t(\mathbf{x})$ for every $\mathbf{x} \in \mathcal{D}(t)$.  A faulty candidate that
differs from $g_t$ on at least one input is accepted only if that input is not in
$\mathcal{D}(t)$.  By Proposition~\ref{thm:witness-soundness}, witnesses are semantically
targeted; the bound follows by counting witnesses.
\end{proof}

\subsection{Repair Monotonicity}
\label{subsec:repair-monotonicity}

\begin{theorem}[Bounded Witness-Coverage Monotonicity]
\label{thm:repair-monotonicity}
Let $\mathrm{Cov}_k$ be the number of witnesses in $\mathcal{D}(t)$ on which the
candidate at iteration $k$ produces correct output.  Then the Semantic Feedback IR
guarantees that the feedback provided to iteration $k+1$ includes every witness failure
from iteration $k$.  Therefore, a repair that fixes the failures from iteration $k$
cannot decrease $\mathrm{Cov}_{k+1}$ below $\mathrm{Cov}_k$.
\end{theorem}
\begin{proof}[Sketch]
The Semantic Feedback IR records all failing witnesses from iteration $k$.  The repair
prompt for iteration $k+1$ includes all failing witness inputs explicitly.  A correct
repair must produce $g_t(\mathbf{x})$ on each presented input; since the inputs are
unchanged, passing them is a necessary condition for the repair to be accepted.
Non-decreasingness of $\mathrm{Cov}$ follows.  Note this is a \emph{coverage} bound, not
a conformance bound: LLM stochasticity may introduce new failures elsewhere; Theorem 3
bounds the resulting FAR.
\end{proof}

\subsection{Prevention-Oriented Metrics}
\label{subsec:metrics}

For mutant set $\mathcal{M}$, prevention detection rate and false-accept rate are:
\begin{equation}
  \mathrm{PDR} = \frac{|\{m \in \mathcal{M} : \neg\mathrm{Accept}(c_m, t_m)\}|}{|\mathcal{M}|},
  \qquad
  \mathrm{FAR} = \frac{|\{m \in \mathcal{M} : \mathrm{Accept}(c_m, t_m) \land \text{faulty}(m)\}|}{|\mathcal{M}|}.
\end{equation}

We report PDR and FAR jointly with strict formal conformance and latency to avoid
over-optimising any single dimension.

\subsection{Full Algorithmic Procedure}
\label{subsec:algorithm}

\begin{algorithm}[t]
\caption{HSP-Agile (\textsc{SgDP} instantiation)}
\begin{algorithmic}[1]
\Require task $t$, budget $K$
\State $\mathcal{D}(t) \leftarrow \mathcal{W}(t)$ \Comment{Constraint-guided witness generation}
\State $\text{feedback}_0 \leftarrow \emptyset$; $\text{best} \leftarrow \mathtt{null}$
\For{$k = 1$ to $K$}
  \State $c_k \leftarrow \mathrm{LLM}(t, \text{feedback}_{k-1})$
  \State $\mathrm{Conf}(c_k,t),\ \text{failures}_k \leftarrow$ eval $c_k$ on $\mathcal{D}(t)$ via oracle $g_t$
  \State $\text{patterns}_k \leftarrow$ pattern guard on $c_k$
  \If{$\mathrm{Accept}(c_k, t)$}
    \Return $c_k$, accepted
  \EndIf
  \State $\text{feedback}_k \leftarrow \{\mathcal{F}(i, c_k, t) : \mathbf{x}_i^* \in \text{failures}_k\}$
  \State $\text{feedback}_k \mathrel{+}= \text{top-}5 \text{ pattern violations from } \text{patterns}_k$
  \State update $\text{best}$ if $\mathrm{Conf}(c_k,t) > \mathrm{Conf}(\text{best},t)$
  \State log $\langle k, \mathrm{Conf}(c_k,t), |\text{failures}_k|, |\text{patterns}_k|\rangle$
\EndFor
\Return $\text{best}$, not-accepted \Comment{Graceful fallback}
\end{algorithmic}
\end{algorithm}

\paragraph{Attempt history logging.}
Every iteration appends $\langle k, \mathrm{Conf}(c_k,t), |\text{failures}_k|,
|\text{patterns}_k|\rangle$ to an \texttt{attempt\_history} record in the output JSONL,
enabling the repair dynamics analysis of E5 and the feedback quality isolation of E6.
