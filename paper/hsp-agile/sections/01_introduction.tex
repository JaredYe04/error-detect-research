\section{Introduction}
\label{sec:intro}

Formal specification languages give software requirements a mathematical precision that
informal descriptions cannot achieve.  Yet this precision creates a new risk when
implementations are generated by large language models (LLMs): the model has no mechanism
to represent or respect the specification's semantic contract, and produces code that is
statistically plausible but formally incorrect.  We call the resulting failure class
\emph{specification-conformance defects} — faults where an LLM-generated implementation
satisfies most inputs yet violates the formal contract at the boundary regions that
formal specifications are specifically designed to characterise.  These defects are
systematically missed by existing test-based repair loops, which sample the input space
without knowledge of specification-level boundary structure.

This paper asks: \emph{is there a general framework that can prevent
specification-conformance defects in LLM-generated code, using the specification itself
as the primary evidence source, without requiring full deductive verification?}

We answer yes, and present the \textbf{Specification-guided Defect Prevention Framework}
(\textsc{SgDP}), a four-component architecture that grounds defect prevention in
specification semantics at every stage: a \emph{Specification Oracle} derived directly
from the formal spec, a \emph{Constraint-guided Witness Generator} that uses SMT solving
to navigate to semantically critical boundary regions, a \emph{Scenario-aware Semantic
Feedback IR} that structures counterexample information at the specification level rather
than the test-case level, and a conjunctive \emph{Acceptance Predicate} that gates release
on both formal conformance and specification-derived quality criteria.

We instantiate \textsc{SgDP} over the SOFL Functional Specification Format (FSF) —
a specification language with ordered, precedence-sensitive scenario semantics used in
Agile-SOFL development~\cite{liu1997sofl,liu2014sofl,li2022agilesofl}.
This instantiation, \textbf{HSP-Agile}, is evaluated
empirically on a benchmark of 120 precedence-sensitive tasks, with experiments ranging
from overall effectiveness to mechanism isolation to specification complexity analysis.
Section~\ref{sec:generalization} further demonstrates the framework on two additional
specification notations, showing that SOFL is not a prerequisite.

The distinction from prior verifier-in-the-loop and CEGIS work is threefold.
First, our Witness Generator is specification-semantics-aware: witnesses are produced by
encoding FSF guard precedence as SMT constraints~\cite{demoura2008z3}, not by random
sampling or symbolic execution on arbitrary postconditions.
Second, our Feedback IR carries specification-level structure — violated scenario identity,
guard predicate text, inferred violation type — that test-feedback loops
discard~\cite{lynn2024verifierloop}.
Third, our evaluation protocol measures \emph{prevention} rather than generation success:
we ask not ``does the LLM produce correct code?'' but ``does the framework prevent
non-conforming code from entering production?''

\paragraph{Contributions.}
\begin{enumerate}
  \item \textbf{Specification-guided Defect Prevention Framework (\textsc{SgDP}).}
    We identify \emph{specification-conformance defects} as a structurally distinct fault
    class and present a general four-component framework addressing it. HSP-Agile
    instantiates \textsc{SgDP} over SOFL/FSF; Section~\ref{sec:generalization} provides
    two additional instantiations.

  \item \textbf{Scenario-aware Semantic Feedback IR.}
    We introduce a typed eight-field intermediate representation (IR) that decomposes a
    conformance failure into violation type, scenario identity, guard text, witness inputs,
    observed vs.\ expected outputs, inferred reason, and suggested fix category.
    Controlled variant comparison (E6) confirms this IR produces faster repair convergence
    than test-case-only or test-plus-expected-output feedback.

  \item \textbf{Constraint-guided Semantic Witness Generation.}
    We formalise the priority formula encoding FSF first-match semantics as Z3 constraints
    and prove (Theorem~\ref{thm:witness-soundness}) that satisfying assignments are
    semantically valid scenario witnesses.  Boundary density experiments (E4) show that
    SMT-based generation maintains scenario coverage across overlap-density regimes where
    random sampling fails by 2--10$\times$.

  \item \textbf{Specification-oriented Prevention Evaluation Protocol.}
    We define a five-dimension evaluation protocol integrating: overall conformance,
    per-operator prevention (PDR/FAR), specification complexity stratification, repair
    trajectory analysis, and feedback quality isolation. The protocol is released as a
    standalone evaluation harness applicable to any specification-guided synthesis system.

  \item \textbf{Open Benchmark and Reproducible Artifact.}
    We release 120 synthetic SOFL/FSF tasks, a 20--30-task real-derived subset from
    HumanEval/MBPP, and 10--20 multi-notation tasks for generalisation evaluation, with
    full experimental logs, analysis scripts, and a reproducibility checklist.
\end{enumerate}
